UEFI

UEFI(Unified Extensible Firmware Interface)는 컴퓨터 시스템의 부팅과 초기화를 담당하는 소프트웨어 인터페이스이다. 기존의 BIOS(Basic Input/Output System)를 대체하는 역할을 한다.

UEFI는 기존의 텍스트 기반 BIOS와 달리 그래픽 사용자 인터페이스(GUI)를 지원한다.

UEFI는 네트워크를 통한 부팅을 지원해 PXE(Preboot Execution Environment)를 사용하여 네트워크를 통해 운영 체제나 다른 부팅 가능한 장치로부터 부트 이미지를 로드할 수 있다.

UEFI는 GPT(GUID Partition Table)를 사용하여 최대 9.4 제타바이트까지의 용량을 지원할 수 있다.

UEFI는 Secure Boot라는 보안 기능을 제공합니다. Secure Boot는 시스템 부팅 과정에서 디지털 서명된 부팅 로더나 운영 체제만을 실행하도록 보장하여 악성 소프트웨어의 실행을 방지한다.

UEFI는 현대적인 컴퓨터 시스템에서 널리 사용되고 있으며, BIOS보다 더욱 많은 기능과 유연성을 제공한다. UEFI는 다양한 운영 체제와 호환되며, 보안과 성능 측면에서 이점을 제공한다.

PXE

PXE(PXE Boot 또는 Preboot Execution Environment)는 네트워크를 통해 컴퓨터를 부팅하는 방법이다. PXE는 컴퓨터의 네트워크 인터페이스를 통해 서버로부터 부트 이미지를 가져와 실행하여 운영 체제나 다른 네트워크 기반의 소프트웨어를 로드한다. PXE를 사용하면 네트워크를 통해 대량의 컴퓨터를 일괄적으로 부팅하고 관리할 수 있다. 대규모 컴퓨터 네트워크나 서버 환경에서 유용하게 사용된다.

-부팅 절차

  1. PXE 클라이언트는 전원을 켠 후 네트워크 인터페이스를 통해 DHCP 서버에 IP 주소를 요청한다.
  2. DHCP 서버는 PXE 클라이언트에게 IP 주소를 할당하고, TFTP 서버의 주소와 부트 이미지 파일의 위치 정보를 제공한다.
  3. PXE 클라이언트는 TFTP 서버로 접속하여 부트 이미지 파일을 다운로드한다.
  4. 다운로드된 부트 이미지 파일은 PXE 클라이언트에 의해 실행되어 운영 체제나 다른 소프트웨어를 로드한다.

WOL과 AC부팅

WOL(Wake-on-LAN)은 네트워크를 통해 원격으로 컴퓨터를 깨우는 기술이다. 일반적으로 컴퓨터는 전원이 꺼져 있는 상태에서는 외부에서의 접근이 불가능하거나 작동하지 않지만 WOL을 사용하면 네트워크 상에서 특정 패킷을 보내어 컴퓨터를 원격으로 켤 수 있다.

AC 부팅(Automatic Power-On)은 컴퓨터의 전원이 꺼진 상태에서 정해진 시간에 자동으로 전원을 켜는 기능이다. 일반적으로 BIOS 설정에서 AC 부팅을 활성화하고, 원하는 부팅 시간을 설정할 수 있다. AC 부팅을 사용하면 예약된 시간에 컴퓨터를 자동으로 켜서 필요한 작업을 수행하거나 원격으로 접속할 수 있다.

WOL과 AC 부팅은 원격 관리, 에너지 절약, 자동화 작업 등 다양한 상황에서 유용하게 사용될 수 있지만 이러한 기능을 사용하기 전에 해당 기능이 지원되는지 확인하고, 설정을 제대로 구성해야 한다. 또한, 보안 상의 이슈를 고려하여 관련된 설정을 적절히 관리해야 한다.

가상머신(네이티브)와 도커

가상머신(VM, Virtual Machine)은 호스트 운영 체제 위에서 독립된 운영 체제 환경을 구축하는 기술이다. 가상머신은 호스트 운영 체제 위에 가상화 소프트웨어(가상머신 모니터, Hypervisor)를 설치하여 가상 하드웨어를 제공한다. 각 가상 머신은 자체 운영 체제, 하드웨어, 네트워크 인터페이스 등을 갖추고 있으며, 다른 운영 체제와 동일하게 작동한다. 가상머신은 여러 개의 가상 머신을 동시에 실행할 수 있으며, 각 가상 머신은 독립적으로 작동하므로 서로 영향을 주지 않는다. 가상머신은 보다 격리된 환경을 제공하지만, 운영 체제 전체를 가상화하기 때문에 오버헤드가 크고 실행 속도가 상대적으로 느릴 수 있다.

도커(Docker)는 컨테이너 기반 가상화 기술을 사용하여 애플리케이션을 실행하는 플랫폼이다. 호스트 운영 체제의 커널을 공유하여 컨테이너를 생성하고 실행한다. 컨테이너는 격리된 환경에서 애플리케이션과 그 종속성을 포함한 실행 환경을 가진다. 가볍고 빠르며 효율적으로 리소스를 사용할 수 있다. 여러 개의 컨테이너를 동시에 실행할 수 있으며, 각 컨테이너는 격리되지만 호스트 운영 체제와 커널을 공유하므로 실행 속도가 가상머신에 비해 빠르다. 또한, 이미지 기반으로 애플리케이션과 종속성을 패키징하고 배포하기 때문에 개발 환경과 운영 환경의 일관성을 유지할 수 있다.

VPN

VPN(가상 사설망, Virtual Private Network)은 인터넷 연결을 통해 개인 정보를 보호하고 온라인 상에서 안전한 통신을 제공하는 기술이다. 사용자의 컴퓨터나 모바일 장치와 VPN 서버 간에 암호화된 연결을 설정하는데 이 연결은 일반적으로 인터넷 서비스 제공자(ISP)나 공공 와이파이 네트워크와 같은 미신뢰할 수 있는 네트워크를 통과하는 동안 데이터의 안전성과 개인 정보 보호를 보장한다.

RDP정의와 공격

RDP(Remote Desktop Protocol)는 원격 데스크톱 연결을 위한 프로토콜이다. 마이크로소프트에 의해 개발되었으며, 사용자가 한 컴퓨터에서 다른 컴퓨터를 원격으로 제어하고 액세스할 수 있도록 한다. RDP를 사용하면 원격으로 접근한 컴퓨터의 데스크톱 화면, 파일 및 리소스를 사용자가 로컬 컴퓨터에서 제어할 수 있다.

-RDP를 통한 공격

  1. 무차별 대입 공격(Brute Force Attack): 공격자는 다양한 사용자 이름과 암호 조합을 시도하여 RDP 로그인에 성공한다. 이를 통해 악성 코드를 설치하거나 시스템을 악용할 수 있다. 강력한 암호 정책, 계정 잠금 정책 등으로 대응할 수 있다.
  2. 취약점 이용 공격(Exploit Attack): RDP 소프트웨어의 취약점을 악용하여 공격자가 원격 시스템에 접근하고 악성 코드를 실행할 수 있다. 시스템을 최신 패치로 유지하고 보안 업데이트를 정기적으로 적용하여 이러한 공격을 방지할 수 있다.
  3. 중간자 공격(Man-in-the-Middle Attack): 공격자는 RDP 연결을 가로채어 중간에 개입하여 사용자의 데이터를 조작하거나 도청할 수 있다. 암호화된 연결을 사용하고 인증서 검증 등의 보안 기능을 활성화하여 이러한 공격을 방지할 수 있다.
  4. 서비스 거부 공격(DoS Attack): RDP 서비스에 대량의 연결 또는 악의적인 요청을 보내어 시스템 리소스를 고갈시키는 공격이다. 적절한 네트워크 보안 장비 및 방화벽을 사용하여 이러한 공격을 차단할 수 있다.

포트포워딩

포트 포워딩(Port Forwarding)은 네트워크에서 인터넷을 통해 특정 포트로 들어오는 트래픽을 특정한 내부 IP 주소와 포트로 전달하는 기술이다. 이를 통해 외부에서 내부 네트워크에 있는 특정 서비스에 접근할 수 있게 된다.

일반적으로 로컬 네트워크에서는 각 장치에 내부 IP 주소가 할당되어 있고 이 장치들은 서로 통신할 수 있지만, 외부 네트워크에서는 이러한 내부 IP 주소에 직접 접근할 수 없다. 하지만 포트 포워딩을 사용하면 외부에서 내부의 특정 포트로 접속하면 해당 포트의 트래픽이 라우터를 통해 내부 IP 주소로 전달된다.

포트 포워딩은 네트워크 보안에 영향을 미칠 수 있으므로, 관련된 보안 상의 고려 사항을 유념해야 합니다. 필요한 경우, 추가적인 보안 조치를 취하여 외부로부터의 악의적인 접근을 방지해야 합니다.

타입스크립트와 자바스크립트

타입스크립트(TypeScript)는 자바스크립트의 상위 집합으로, 정적 타입을 지원하여 개발자가 코드를 더 안정적으로 작성하고 유지할 수 있도록 도와준다.

변수, 매개변수, 함수 등에 명시적인 타입을 지정할 수 있어 개발자가 코드를 작성할 때 타입 오류를 미리 찾을 수 있고, 코드의 가독성과 유지 보수성을 향상시킨다. 컴파일러가 타입을 검사하고 오류를 보고해주기 때문에 런타임에서 발생할 수 있는 많은 버그들을 사전에 방지할 수 있다.

기존의 자바스크립트 코드를 그대로 사용하거나, 자바스크립트 라이브러리와 프레임워크를 쉽게 통합하여 사용할 수 있다. 개발자 도구와의 통합을 강화하기 위해 다양한 도구와 에디터 확장을 제공한다. 타입스크립트 컴파일러를 사용하여 자바스크립트 코드로 변환할 수 있으며, 대부분의 개발 환경에서 타입스크립트를 지원하고 있다.

자바스크립트(JavaScript)는 웹 페이지를 동적으로 만들기 위해 사용되는 스크립트 언어이다. 웹 브라우저에서 실행되며, 클라이언트 측 스크립트 언어로서 HTML과 함께 사용되어 웹 페이지의 동작을 제어하고 상호작용성을 부여한다. 웹 페이지의 요소들을 선택하고 조작할 수 있으며, 이벤트 처리, 폼 검증, 애니메이션, 데이터 통신 등 다양한 기능을 구현할 수 있다.

자바스크립트는 동적 타입 언어이며, 변수의 타입을 명시적으로 선언하지 않는다. 또한, 객체 지향 프로그래밍 및 함수형 프로그래밍 개념을 지원한다.

자바스크립트는 다양한 브라우저에서 지원되며, 대부분의 웹 애플리케이션에서 필수적으로 사용된다. 또한, 자바스크립트는 라이브러리와 프레임워크를 통해 개발 생산성을 높이고 코드 재사용성을 높일 수 있다. 대표적인 자바스크립트 라이브러리로는 jQuery, React, Vue.js 등이 있으며, 프레임워크로는 Angular, Node.js 등이 있습니다.

Graph-QL과 REST-API

GraphQL은 페이스북에서 개발된 쿼리 언어이다. 웹 클라이언트가 데이터를 서버로부터 효율적으로 요청하고 응답받을 수 있게하는 기술이다. 기존의 RESTful API와 비교하면 GraphQL은 클라이언트가 필요로 하는 데이터를 정확하게 요청하고 응답받을 수 있는 유연한 방식을 제공한다.

GraphQL은 API의 유연성과 확장성을 제공한다. 클라이언트는 필요한 데이터를 직접 지정할 수 있고, 서버는 요청에 따라 필요한 데이터만 반환할 수 있습니다. 이는 클라이언트와 서버 간의 밀접한 협력을 가능하게 하며, API의 변경 없이 클라이언트의 요구 사항에 맞춰 데이터를 가져올 수 있다.

또한, 다양한 데이터 소스를 통합하여 하나의 API로 제공할 수 있다. 여러 개의 마이크로서비스나 데이터베이스에서 데이터를 가져올 때도 GraphQL을 사용하여 한 번의 요청으로 필요한 데이터를 효율적으로 가져올 수 있다.

REST API(Representational State Transfer)는 웹 서비스 아키텍처 스타일 중 하나로 클라이언트와 서버 간의 통신을 위해 HTTP 프로토콜을 기반으로 사용되는 API이다.

REST API는 널리 사용되는 API 디자인 스타일 중 하나이며, 웹 서비스 개발에서 많이 활용된다. REST의 원칙과 제약 조건을 따르면서 리소스 중심의 API를 설계하고 구현함으로써 유연하고 확장 가능한 시스템을 구축할 수 있다.

GraphQL은 클라이언트가 특정한 데이터 요구사항을 가지고 있고, 유연성과 효율성이 필요한 경우에 유용하며, REST API는 간단하고 표준화된 인터페이스를 필요로 하는 경우에 더 적합하다.

UI/UX

UI(User Interface)는 사용자가 제품이나 서비스와 상호 작용하는 방법을 의미한다. 화면 구성, 버튼, 메뉴, 레이아웃 등 사용자가 제품이나 서비스에 접근하고 사용할 수 있도록 하는 모든 시각적 요소를 포함한다. UI의 목적은 사용자가 제품이나 서비스를 사용하는 것이 쉽고 편리하도록 하는 것이다.

UX(User Experience)는 사용자가 제품이나 서비스와 상호 작용하는 동안 얻는 총체적인 경험을 의미한다. UX는 UI뿐만 아니라 제품이나 서비스의 사용성, 디자인, 기능, 성능 등 모든 요소를 포함한다. UX의 목적은 사용자가 제품이나 서비스를 사용하는 동안 만족하고 즐거운 경험을 할 수 있도록 하는 것이다.

UI와 UX를 잘 디자인하기 위해서는 사용자의 요구 사항을 이해하는 것이 중요하다. 사용자의 요구 사항을 이해하면 사용자에게 최적화된 UI와 UX를 디자인할 수 있다.
